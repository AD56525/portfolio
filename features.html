про які корисно знати кожному Java / C-розробнику

На JavaScript вплинули багато мов, при розробці була мета зробити мову схожою на Java. JavaScript має низку властивостей об'єктно-орієнтованої мови, але реалізоване в мові прототипування обумовлює відмінності в роботі з об'єктами в порівнянні з традиційними мовами.

Особливості

Незважаючи на схожість назв, мови Java та JavaScript є двома різними мовами, що мають відмінну семантику, хоча й мають схожі риси в стандартних бібліотеках та правилах іменування.

Іноді JavaScript може вводити розробника в оману, а іноді - доводити до сказу через свою неповної консистентності. Є в JavaScript деякі речі, які тільки заплутують і збивають з пантелику. Найвідоміші з них оператор with , неявні глобальні змінні і дивна поведінка при операції порівняння.

Мабуть, навколо JavaScript розгоралося найбільше суперечок в історії програмування. Крім його недоліків (частково розглянутих в нових специфікаціях ECMAScript), більшість програмістів незадоволені такими моментами:

• DOM, який багато хто помилково вважає еквівалентом самої мови JavaScript, володіє дуже невдалим API.

• Коли переходиш на JavaScript з мов С і Java, то потрапляєш в пастку синтаксису, який влаштований не так, як в імперативних мовах. Це дуже часто призводить до багів і сильно дратує.

В результаті JavaScript знайшов досить погану репутацію, якою він, в общем-то, не заслуговує. І найчастіше це пов'язано з тим, що багато розробників переносять на JavaScript свій досвід роботи на Java або С / С ++. Тут розібрані три найбільш важких випадку, демонструють різницю в підходах між Java і JavaScript.
